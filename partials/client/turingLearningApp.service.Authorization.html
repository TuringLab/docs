Authorization
service in module turingLearningApp

Description

The Authorization service can be used to authorise a state change. It is assumes that user roles are ordered and that higher users can access routes for users with lower privileges.

Dependencies

  • $state
  • $rootScope
  • Principal

Methods

  • authorize()

    Authorize state change to check that the currently logged in user is able to access a particular route. This is done by comparing the index of the user role against next state role

    Returns
    Promise
